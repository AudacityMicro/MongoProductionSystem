04

CNC telemetry

PathPilot/LinuxCNC status is collected over SSH. The Debugging page can also run only the specifically assigned Mill program buttons.

The controller must allow SSH access for this LinuxCNC user. Telemetry uses linuxcnc.stat().poll(); assigned Mill Debug buttons use linuxcnc.command() to load and start only their configured file.

Default: 16. Occupied positions must be cleared before reducing this number below them.

05

Robot file access

Use SFTP to browse program files stored on the robot controller.

Robot extensions filter the controller-program button chooser; .urp is the default. Universal Robots defaults are preconfigured: root / easybot. The password is shown and editable here, and is returned to browsers that can access this trusted local-network application.

06

Debugging controls

Control diagnostic access and configure the Robot and Mill program buttons shown on the Debugging page.

Each button has its own display name, controller filename, and color. Reducing a count hides higher-numbered buttons without deleting their saved configuration.

07

Robot Programs page

Manage controller files through SFTP and optionally open downloaded files in a local editor.

The command runs on this server computer with a downloaded copy of the selected controller file. Default: code for Visual Studio Code.

08

Mill Programs page

Manage PathPilot user program files through the configured CNC SSH connection.

Cycle results archive

PathPilot normally writes RESULTS.TXT beside the Gcode folder. Both paths must stay inside the PathPilot controller data directory. Archives use program__UTC-timestamp__RESULTS.TXT filenames; the archive directory is created automatically.

Uses the PathPilot host, SSH port, username, password, and timeout above. All file operations stay inside this program directory. Default: /home/operator/gcode/Gcode.

09

Robot pallet motion

Configure safe motion geometry and generated programs for Pool positions and manual mill transfers.

Mill door and Erowa commands

Assign the robot outputs used for each command. A momentary command returns the output to its opposite state after 0.25 seconds; a maintained command leaves it in the selected state.

Pallet tool orientation
Intermediate joint-safe poses

Create ordered, named joint configurations and assign each to one or more Pool positions. Assigned poses are traversed after the shared safe pose and again in reverse before the final return.

New intermediate joint-safe poseCapture a physically verified J0-J5 robot posture, then assign the Pool positions that must travel through it.
Saves these settings, writes local .script files, then synchronizes the same files to the robot.

The shared safe pose and intermediate safe poses are explicit J0-J5 joint configurations. Capture them from physically verified robot postures; generated scripts use joint-space movej transitions and never infer an arm configuration from Cartesian coordinates. Pool pick and place scripts use the Y approach clearance. Mill load and unload scripts use the dedicated X approach clearance. The pallet and mill entry poses remain Cartesian for controlled linear moves near fixtures.
